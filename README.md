# 应急知识图谱智能问答系统

## 项目简介

应急知识图谱智能问答系统是一个基于知识图谱和大语言模型（LLM）的智能问答平台，旨在为用户提供精准、实时的应急知识查询服务。系统通过动态构建知识图谱、实体关系抽取和多路检索策略，实现了应急知识的高效管理与智能问答。

## 功能特性

- **动态知识图谱构建**：从用户对话中自动抽取实体与关系，实时扩展知识图谱
- **多路检索问答**：融合Cypher查询、实体识别和上下文关联，提升问答准确率
- **自然语言交互**：支持自然语言对话，低门槛获取应急知识
- **实时响应**：优化后响应时间≤1秒，满足实时交互需求
- **跨域访问**：支持前端跨域请求，适配多端访问

## 技术栈

- **前端**：Umi.js
- **后端**：Flask、Python
- **数据库**：Neo4j图数据库
- **大语言模型**：Qwen-Long（通过DashScope API）
- **其他**：py2neo（Neo4j Python驱动）、Flask-CORS（跨域支持）

## 环境要求

- Python 3.7+
- Neo4j 4.0+
- DashScope API Key（用于调用Qwen-Long模型）

## 安装部署

### 1. 安装依赖

```bash
# 安装Python依赖
pip install flask flask-cors py2neo dashscope
```

### 2. 配置Neo4j数据库

1. 安装并启动Neo4j数据库
2. 创建数据库用户
3. 确保Neo4j服务运行在 `127.0.0.1:7687`

### 3. 配置DashScope API Key

在 `app.py` 文件中替换为你的DashScope API Key：

```python
# app.py 第188行
dashscope.api_key = '你的API Key'
```

### 4. 启动服务

```bash
# 进入项目目录
cd extra_kg_chat

# 启动后端服务
python app.py

# 前端访问地址
# 打开浏览器访问：http://localhost:5000/chat/index.html
```

## 使用方法

1. 在前端页面输入应急相关问题，如：
   - "火灾发生时应该如何逃生？"
   - "地震的应急措施有哪些？"
   - "灭火器的使用方法是什么？"

2. 系统会自动分析问题，从知识图谱中检索相关信息，并结合大语言模型生成精准回答。

3. 系统会根据对话内容自动扩展知识图谱，持续丰富应急知识体系。

## 核心功能模块

### 1. 知识图谱构建模块

- 设计了12+实体类型（如应急手册、具体事故、危险源等）和9+关系类型（如包含、需要、涉及等）
- 支持从用户对话中自动抽取三元组并动态扩展图谱

### 2. 多路检索问答模块

- 融合Cypher语句生成、实体识别和上下文关联的三重检索机制
- 实现精准匹配用户问题与知识图谱内容

### 3. 前端交互模块

- 基于Umi框架实现实时对话界面
- 支持历史消息展示与清空功能
